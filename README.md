## ­ЪїЪ Branch Workflow

### Branch Structure

- **`master`**: Production deployment branch for GitHub Pages
  - Contains only stable, tested, and completed code
  - All final projects and completed modules
  - Automatically deployed to GitHub Pages

- **`beta`**: Internal deployment branch for testing
  - Used for testing combined features or multiple changes
  - Integration testing of completed modules
  - Preview environment for stakeholders

- **`develop`**: Main development branch
  - Active development of course modules
  - Feature integration and testing
  - Base branch for all feature branches

### Feature Branch Naming Convention
- `feature/module-{number}-{topic}` (e.g., `feature/module-1-html-basics`)
- `exercise/module-{number}-{exercise-name}` (e.g., `exercise/module-2-responsive-design`)
- `project/milestone-{number}` (e.g., `project/milestone-1-portfolio`)

## ­Ъџђ Getting Started

### Prerequisites
- Text editor (VS Code recommended)
- Git installed on your machine
- GitHub account
- Modern web browser

### Quick Setup

1. **Clone the Repository**
   ```bash
   git clone https://github.com/GhagSagar23/vue_js_udemy.git
   cd vue_js_udemy
   ```

2. **Create Development Branch**
   ```bash
   git checkout -b develop
   ```

3. **Start with Module 1**
   ```bash
   cd modules/01-html
   # Follow the README.md in the module folder
   ```

4. **Enable GitHub Pages**
   - Go to repository Settings
   - Scroll to Pages section
   - Set source to `master` branch
   - Your progress page will be available at `https://GhagSagar23.github.io/vue_js_udemy`
